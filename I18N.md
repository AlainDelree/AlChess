# AlChess — Guide d'internationalisation (i18n)

## Vue d'ensemble

AlChess supporte plusieurs langues via un système de fichiers JSON et un module JS léger.
La langue est sélectionnée dans l'interface et sauvegardée en `localStorage`.

**Langues disponibles :**
- 🇫🇷 Français (`fr`) — langue de référence
- 🇬🇧 English (`en`)

---

## Architecture

```
nicsoft/web/static/
  i18n/
    fr.json         — Traductions françaises (source de vérité)
    en.json         — Traductions anglaises
    <code>.json     — Ajouter ici pour une nouvelle langue
  i18n.js           — Module i18n (chargement, t(), applyToDOM())
```

Le module `i18n.js` est chargé dans `index.html` avant `app.js`.
Il expose une fonction globale `t("clé")` utilisée partout dans `app.js`.

---

## Convention de nommage des clés

Format : `section.sous-section.element`

| Préfixe       | Contenu                                      |
|---------------|----------------------------------------------|
| `common.*`    | Chaînes partagées (Retour, OK, Annuler…)     |
| `menu.*`      | Écran menu principal                         |
| `config.*`    | Écrans de configuration (pédagogique, HH)    |
| `game.*`      | Panneau de jeu en cours                      |
| `feedback.*`  | Qualité des coups (bon, imprécision…)        |
| `labo.*`      | Écran Laboratoire                            |
| `exercices.*` | Écran Exercices                              |
| `retrans.*`   | Écran Retranscription                        |
| `outils.*`    | Écran Outils Exercices                       |
| `modal.*`     | Boîtes de dialogue modales                   |
| `toast.*`     | Notifications toast                          |
| `error.*`     | Messages d'erreur (board, réseau…)           |
| `server.*`    | Clés envoyées par le serveur Python          |

**Exemples :**
```
menu.btn.pedagogique        → "Mode Pédagogique" / "Training Mode"
feedback.bon                → "Bon coup" / "Good move"
toast.copie                 → "Copié !" / "Copied!"
error.board.non_detecte     → "Échiquier non détecté…" / "Board not detected…"
server.popup.mat            → "♚ Échec et mat !" / "♚ Checkmate!"
```

### Règles
- Tout en minuscules, séparateurs `.` et `_` uniquement
- Pas d'espaces, pas d'accents dans les clés
- La clé doit décrire le sens, pas le texte exact (`feedback.bon` pas `feedback.bon_coup`)
- Si une chaîne est identique dans plusieurs contextes → clé `common.*`

## Interpolation de variables

Pour les messages avec valeurs dynamiques, utiliser `{variable}` :

```json
// fr.json
{ "error.ouverture_introuvable": "Ouverture '{id}' introuvable." }

// en.json
{ "error.ouverture_introuvable": "Opening '{id}' not found." }
```

```js
// Dans app.js
t("error.ouverture_introuvable", { id: ouverture_id })
```

## Utilisation dans le HTML

Ajouter l'attribut `data-i18n` sur l'élément porteur du texte.
`applyToDOM()` remplace automatiquement le `textContent` à chaque changement de langue.

```html
<!-- Texte simple -->
<button data-i18n="menu.btn.pedagogique">Mode Pédagogique</button>

<!-- Placeholder d'input -->
<input data-i18n-placeholder="outils.san.placeholder" placeholder="Collez votre PGN ici…">

<!-- Attribut title (tooltip) -->
<button data-i18n-title="common.btn.copier" title="Copier">📋</button>
```

**Règle** : conserver le texte français en dur dans le HTML comme fallback visible
si le JSON n'est pas encore chargé.

---

## Utilisation dans app.js

Remplacer toute chaîne visible par `t("clé")` :

```js
// Avant
afficherToast("Copié !", "success");

// Après
afficherToast(t("toast.copie"), "success");
```

Pour le HTML généré dynamiquement :
```js
// Avant
html += `<td>Bon coup</td>`;

// Après
html += `<td>${t("feedback.bon")}</td>`;
```

## Messages du serveur Python

Le serveur envoie des **clés** (pas du texte) dans les événements socket.
Le JS traduit côté client avec `t(key)`.

```python
# alchess.py — Avant
send_event("board_error", {"message": "Échiquier non détecté."})

# Après
send_event("board_error", {"key": "error.board.non_detecte"})
```

```js
// app.js — handler
socket.on("board_error", (data) => {
    afficherErreur(t(data.key));
});
```

**Liste des événements Python qui envoient des clés :**
- `board_error` → `data.key`
- `popup` → `data.key`
- `undo_move` → `data.message_key` (optionnel, si message visible)
- `position_error` → pas de message visible côté client (géré par position_ok/position_error)

## Ajouter une nouvelle langue (ex: Espagnol)

1. **Copier** `en.json` → `es.json`
2. **Traduire** toutes les valeurs (ne pas toucher aux clés)
3. **Ajouter** la langue dans le sélecteur HTML :

```html
<!-- index.html — dans #lang-selector -->
<option value="es">🇪🇸 ES</option>
```

4. **Ajouter** la langue dans `i18n.js` :

```js
const SUPPORTED_LOCALES = ['fr', 'en', 'es'];
```

5. **Tester** : lancer l'appli, sélectionner ES, vérifier visuellement chaque écran.

C'est tout. Aucune modification du code JS ou Python n'est nécessaire.

---

## Ajouter une nouvelle chaîne traduite

1. Ajouter la clé dans **tous** les fichiers JSON existants (`fr.json`, `en.json`, …)
2. Ajouter `data-i18n="clé"` sur l'élément HTML (si statique)
   **ou** utiliser `t("clé")` dans `app.js` (si dynamique)
3. Si le texte vient du serveur Python : envoyer la clé, pas le texte

**Ne jamais** ajouter une clé dans un seul fichier JSON — toutes les langues doivent avoir la clé, même si la traduction est temporairement en anglais.
